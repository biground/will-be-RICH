# 🚀 量化回测系统 (Will-Be-RICH Backtest Platform)

一个功能完整的**三阶段智能回测平台**，支持 ETF 基金和 A 股指数的量化策略开发与验证。采用 Streamlit 构建交互式 UI，集成 VectorBT、AKShare、Plotly 等行业领先工具，提供专业级的风险分析与参数优化能力。

---

## ✨ 核心特性

### 🔬 三阶段回测流程
- **阶段一**：单指标测试 - 独立评估 20+ 技术指标（MA、RSI、MACD 等）
- **阶段二**：组合优化 - 对高分指标进行 AND/OR 逻辑组合回测（自动产生数百个组合）
- **阶段三**：参数扫描 - 针对最优组合进行滑动窗口、参数范围的敏感性分析

### 📊 专业量化分析
- **策略评级系统**（A-F 等级）：基于 6 个加权维度的综合评分
- **雷达图展示**：6 维度对标分析（收益风险比、回撤控制、胜率、盈亏比、盈利因子、卡玛比率）
- **净值与回撤曲线**：双 Y 轴图表，直观展示策略表现与最大回撤
- **滚动夏普比率**：60 日窗口的稳定性分析
- **收益分布直方图**：实际分布与正态分布对标，评估尾部风险
- **年度收益拆解**：按年份、月份热力图展示盈亏分布
- **连续盈亏统计**：最大连胜/连亏笔数，衡量策略心理压力
- **基准对标**：与指数 Buy-and-Hold 策略的全方位对比

### 🔐 OTP 邮件验证
- 防止他人滥用系统：每次会话首次运行回测需要邮件验证
- 6 位随机验证码，5 分钟有效期，5 次尝试上限
- 本次会话验证一次后无需重复，关闭浏览器标签页后失效

### 🎛️ 灵活的参数配置
- **标的选择**：ETF 基金 / 指数双模式
- **常用快选**：预置沪深 300、中证 500、创业板等常见标的
- **自定义代码**：精确控制标的、基准指数、回测周期
- **数据频率**：日线、周线可选
- **费率设置**：佣金、滑点、印花税完全自定义
- **复权方式**：前复权、后复权、不复权
- **阶段二筛选**：TOP N 策略个数、最低夏普门槛、最大回撤上限可调

### 🌐 智能数据获取
- **多源 Fallback**：失败自动切换 7 层数据源（东财、腾讯、新浪、中证指数官网等）
- **本地缓存**：减少重复请求，支持跨日期范围缓存复用
- **网络代理支持**：内置直连 / 本地代理（Clash / V2Ray）双模式切换
- **代理自动降级**：用户设置proxy 失败时自动回退直连

### 📱 现代化 UI/UX
- **固定顶部导航栏**：4 页面快速切换，始终可见
- **响应式多列布局**：PC 端自适应，侧边栏配置区独立
- **直观的参数说明**：每个设置都有详细的 hover 提示与参考标准
- **实时进度反馈**：进度条 + 状态文本，回测过程透明可见

### 🏆 Manual Phase-3 自选模式
- **两步式工作流**：先完成阶段一、二，查看全部候选策略
- **自主选择**：从第一步结果中选择感兴趣的策略，而非强制采用夏普最高的组合
- **个性化优化**：针对特定策略进行深度参数扫描

## 🚀 快速开始

### 前置要求
- **Python 3.12+**
- **pip** 或 **uv**

### 1️⃣ 克隆和依赖安装

```bash
# 克隆项目
git clone <repo-url>
cd will-be-RICH

# 安装依赖
pip install -r requirements.txt

# 或使用 uv（更快）
uv pip install -r requirements.txt
```

### 2️⃣ 配置邮件验证（可选但推荐）

在 `.streamlit/secrets.toml` 中添加 SMTP 配置：

```toml
[smtp]
host     = "smtp.163.com"
port     = 465
user     = "your-email@163.com"
password = "your-auth-code"      # 163 授权码，非用户密码
receiver = "your-email@163.com"
```

> **提示**：
> - 163 邮箱免费授权码获取：账户设置 → POP3/SMTP/IMAP → 开启并获取权限码
> - `.streamlit/secrets.toml` 已被 `.gitignore` 保护，不会提交到版本库

### 3️⃣ 启动应用

```bash
# 标准启动（文件变化不自动重载）
streamlit run app.py

# 本地开发模式（保存文件自动刷新）
streamlit run app.py --server.fileWatcherType auto

# 指定端口
streamlit run app.py --server.port 8501
```

应用将在 `http://localhost:8501` 打开。

### 4️⃣ 首次使用流程

1. 进入【🚀 新建回测】页面
2. 在侧边栏配置：标的、时间范围、费率、数据频率等参数
3. 选择【阶段三优化目标】：
   - **自动**：一键完成三阶段，自动选夏普最高的策略进行参数优化
   - **手动选择**：分两步 – 先运行阶段一、二看结果，再自选策略进行阶段三
4. 点击【运行】按钮 → 邮件验证 → 开始回测

## 📖 功能详解

### 🔹 阶段一：单指标测试

自动生成 20+ 种技术指标的交易信号，每个指标独立回测：

| 指标 | 说明 |
|------|------|
| MA（移动平均） | 快线/慢线交叉策略 |
| RSI（相对强弱） | 超买/超卖反转策略 |
| MACD（指数平滑） | 零轴交叉、直方图转向等 |
| KDJ（随机指标） | K/D 线交叉 |
| Bollinger（布林带） | 上下轨突破 |
| ADX（趋势指标） | 趋势强弱判断 |
| ... 等 20+ 更多 |

**输出**：按夏普比率排序的单指标表格，展示年化收益、最大回撤、胜率、交易次数等核心指标。

### 🔹 阶段二：组合优化

自动对阶段一的 Top N 指标进行两两组合，支持 AND/OR 逻辑：

- **AND 逻辑**：买入信号同时出现才进场（更保守，交易次数少，单笔收益高）
- **OR 逻辑**：任何信号出现即进场（激进，交易频繁，分散风险）

**产出量**：N=10 时约 90 个 AND + 90 个 OR = 180 个组合，按夏普排名。

### 🔹 阶段三：参数扫描

对最优组合进行滑动窗口敏感性分析：

- **快线周期** / **慢线周期** / **平滑系数** 等参数
- 生成参数×参数的结果矩阵，找出最优参数组合

**应用场景**：
- 评估策略对参数的敏感度（稳健性）
- 找到本历史周期内的最优参数（避免过度拟合风险）

### 🔹 OTP 邮件验证

保护系统资源防止滥用：

```
用户点击【发送验证码】
    ↓
系统生成 6 位随机数字（如 382641）
    ↓
通过 163 SMTP SSL 发送邮件到绑定邮箱
    ↓
用户输入验证码 + 点【验证并解锁回测】
    ↓
比对成功 → session_state._otp_verified = True
    ↓
本会话可无限次运行回测（不再提示验证）
    ↓
关闭浏览器标签页 → 失效，下次登陆需重新验证
```

**验证码有效期**：5 分钟，UI 实时显示剩余秒数  
**错误尝试上限**：5 次，超出后需重新发送

### 🔹 数据管理

#### 本地缓存

回测数据存储在 `data/cache/` 下按规则命名：
```
{symbol}_{start_date}_{end_date}_{adjust}.csv   # ETF 缓存
{symbol}_{start_date}_{end_date}_index.csv      # 指数缓存
```

**缓存复用策略**：
- 同一 symbol 不同时间范围 → 自动检测任意已有缓存，截取所需日期区间
- 避免重复下载，加快回测启动

#### 数据库存储

所有回测结果保存到 SQLite 数据库 `backtest_results.db`，包含：

| 表 | 用途 |
|-----|-----|
| `runs` | 回测任务元数据（name、ETF、时间、参数） |
| `phase1_results` | 阶段一结果（单指标排名） |
| `phase2_results` | 阶段二结果（组合策略排名） |
| `phase3_results` | 阶段三结果（参数扫描矩阵） |
| `best_strategy` | 最优策略详情（净值、月度收益热力图、交易记录） |

**回溯能力**：支持查看并重加载任何历史回测的完整参数，一键复现。

## 🔧 故障排查

### 问题 1：邮件发送失败

**症状**：点击【发送验证码】报错 "Connection refused" 或超时

**排查步骤**：
1. 检查 `.streamlit/secrets.toml` 配置正确（host、port、用户名、授权码）
2. 确认 163 邮箱已开启 SMTP 服务并获取授权码（非用户密码）
3. 检查防火墙是否阻止 465 端口出站连接
4. 尝试用办公网络或 VPN（可能是 ISP 限制）

**解决**：
```python
# 临时禁用 OTP 验证进行开发测试
# pages/1_新建回测.py 第 ~1070 行注释掉：
# if not _otp_gate():
#     st.stop()
```

### 问题 2：数据获取失败

**症状**：选择 ETF 后报 "所有数据源均不可用"

**原因**：
- ETF 代码错误（应为 6 位数字，如 510300）
- AKShare 数据源完全下线（罕见）
- 网络代理配置错误

**排查步骤**：
1. 检查代码格式（6 位数字，无其他符号）
2. 切换标的【直连（绕过代理）】或【使用本地代理】试试
3. 查看终端日志了解具体失败的数据源和错误信息

### 问题 3：Streamlit 文件变化不热加载

**症状**：修改 Python 代码保存后，页面仍未更新

**原因**：`.streamlit/config.toml` 设 `fileWatcherType = "none"`

**解决**：
```bash
# 方式 1：启动参数覆盖
streamlit run app.py --server.fileWatcherType auto

# 方式 2：浏览器 R 键手动刷新
# 在 Streamlit 网页上按 R，立即重载

# 方式 3：修改 config.toml
# 将 fileWatcherType = "none" 改为 "auto"
# 但云部署时需改回 "none" 以节省资源
```

### 问题 4：回测耗时过长

**症状**：阶段一运行 5+ 分钟都没有结果

**常见原因**：
- 指标个数过多（共 ~20 个）
- 数据周期过长（10+ 年）
- 硬件配置较低

**优化建议**：
1. 减少回测周期（如 2018–2023 改为 2020–2023）
2. 改用周线而非日线（4-5 倍加速）
3. 增加阶段二 TOP N 筛选门槛（减少阶段二/三的组合数）

## 🎯 使用场景示例

### 场景 1：评估均线策略

1. 【标的】选择 510300（沪深300ETF）
2. 【周期】2015-01-01 ~ 2023-12-31（见证一次完整牛熊）
3. 【频率】日线
4. 【费率】佣金 0.02%, 滑点 0.1%, 印花税 0（ETF 不征）
5. 【优化】自动（一键结束）
6. 查看【阶段一】MA 指标表现 → 选出最优的均线组合
7. 在【最优策略】标签页查看净值曲线、月度热力图、交易记录

**预期结果**：很多均线组合都会盈利（因为 2015-2023 大趋势向上），但夏普比率差异很大。

### 场景 2：定制多指标组合

1. 【优化】选择"手动选择"
2. 点【运行阶段一 & 二】
3. 浏览【阶段一】top 5 指标和【阶段二】top 10 组合
4. 看到 "RSI + MACD [AND]" 很有趣，选择它
5. 点【继续阶段三】
6. 阶段三对该组合的参数进行 5×5 矩阵扫描，找最优参数

**优势**：人工审视中间结果，避免数据过拟合陷阱。

### 场景 3：对标基准收益

在【最优策略】下的【基准对比】栏查看：
- 策略年化收益 15% vs 基准 8% → 策略超额收益 7%
- 策略最大回撤 25% vs 基准 35% → 风险更低
- 策略夏普 1.2 vs 基准 0.6 → 单位风险收益更高

**启示**：策略战胜基准的核心是风险调整后收益，不是绝对收益。
